Backend Web Application with Kotlin

Sep 9, 2024

Kotlin is a concise and versatile JVM based language created by JetBrains. I used Kotlin for developing many different services and backend web applications.

Kotlin is a statically-typed programming language and is fully interoperable with Java [1]. The build tools Maven and Gradle fully support Kotlin, and these tools help managing the configuration of the libraries and the build process of the application.

Kotlin provides many features and concepts, such as null safety and extension functions, and it allows software developers to create concise and expressive code.

In this article I will describe some technical aspects of the Active recall learning web application that I developed using Spring Boot 2 and Kotlin.

Spring Boot and Kotlin make it really easy to create Rest controllers, services, entities, and interacting with the database.

The following is an example of a Rest Controller:

@RestController
@RequestMapping("/api/workspaces")
class WorkspaceController(private val workspaceService: WorkspaceService) {
  @PostMapping(consumes = [MediaType.APPLICATION_JSON_VALUE])
  fun create(
    @RequestBody request: WorkspaceCreateRequest
  ): ResponseEntity<Workspace> {
    val workspace = workspaceService.create(request)
    val builder = ResponseEntity.status(HttpStatus.CREATED)
    builder.location(URI("/api/workspaces/${workspace.id}"))
    return builder.body(workspace)
  }
}

The following is an example of a Service:

@Service
class WorkspaceService(
  private val workspaceRepository: WorkspaceRepository,
  private val workspaceIdGenerator: WorkspaceIdGenerator
) {
  fun create(workspaceCreateRequest: WorkspaceCreateRequest): Workspace {
    val workspaceId = workspaceIdGenerator.create()
    val workspace = Workspace.create(workspaceId, workspaceCreateRequest)
    val entity = WorkspaceEntity.create(workspace)
    return Workspace.create(workspaceRepository.save(entity))
  }
}

The following is an example of an Entity, defined as a data class:

@Entity
@Table(name = "workspaces")
data class WorkspaceEntity(
  @Id val id: String = "",
  val name: String = ""
) {
  companion object {
    fun create(workspace: Workspace) : WorkspaceEntity {
      return WorkspaceEntity(workspace.id, workspace.name)
    }
  }
}

The following extension function is used for converting a Java Optional<T> to a Kotlin nullable T:

import java.util.Optional
fun <T : Any> Optional<T>.toNullable(): T? = this.orElse(null)

The business logic of the application is simple to follow, and I found the code written in Kotlin easy to read. The features provided by the Kotlin language, such as data classes, default arguments, and expressions, help to reduce the boilerplate code.

I used Spring Security with JWT for managing the user authentication and authorization in the web application. I created the REST API documentation with OpenAPI, compliant with Swagger and Postman.

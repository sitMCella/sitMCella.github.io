Story Outlining Web Application

Sep 2, 2024

I developed a story outlining web application. A story outline defines the roadmap for a story, and provides a structure for writing a manuscript, guiding the writer from the beginning to the end.

Some of the components of a story outline are the characters definition, the plots, and the scenes. A plot consists of the sequence of events that constitute the story. A scene instead is composed of a series of continuous (self-contained) events, and is part of a plot. The scenes of a story can also be grouped in different chapters of the manuscript.

The web application represents a book as a Project. The project consists of a title, a description, and a list of Tags. The Tags are configured at Project level, in order to maintan consistency over the Tags assigned to the scenes of the story. You can decide for example to create one Tag for every character, and later on track on which scenes each character appears throughout the story.

I decided to provide two different structures for the story outline in the Project Map page and the Project Book page.

The Project Map page defines a storyboard that collects the scenes in a tabular format with the chapters as columns and the plots as rows. The scenes are cards that shows the extract of the scene, and can be expanded to get access to the content and the configuration of the scene. The Project Map page allows you to plan and structure the scenes in plots and chapters. In addition you can assign Tags to the scenes and filter the scenes by Tags.

The Project Book page instead shows the list of scenes. In this page the scene editor is always visible, but you can't configure the scenes or filter the scenes by Tag.

Web Application Frontend in Vue.js

Sep 1, 2024

I decided to explore the Vue framework because I was interested to understand what could have been the effort to create a project from scratch without prior knowledge on the platform.

In this post I will introduce the Vue framework and I will discuss some features that can speed up approaching Vue, in particular for people that have experience working with React. It's not scope of this article to compare Vue with other frameworks.

Vue was initially designed for rapid prototyping of web applications, but is well suited for developing large-scale rective web applications. The command npm create vue@latest scaffolds a new Vue application. The core features of Vue are declarative rendering and reactivity. The following are the definitions from the Vue documentation [1]:

Declarative Rendering: Vue extends standard HTML with a template syntax that allows us to declaratively describe HTML output based on JavaScript state.

Reactivity: Vue automatically tracks JavaScript state changes and efficiently updates the DOM when changes happen.

Vue uses an HTML-based template syntax that allows you to declaratively bind the rendered DOM to the underlying component instance's data. The single-file component (SFC) format encapsulates the component's logic (JavaScript), template (HTML), and styles (CSS) in a single .vue file.

Both the React and Vue frameworks share some concepts: Virtual DOM, component-based UI, routing, and state management. Some of the most interesting features of Vue are the Composition API, the lifecycle hooks, and Typescript support.

The Composition API structures the component logic in functions, and is typically used with <script setup> for the single-file component format. The functions handle the reactive state variables and can be combined to create complex scenarios.

The lifecycle hooks are functions that can be used to add your own code at specific stages of the initialization of the component.

Vue provides first-class Typescript support. Typescript brings type safety on top of the Javascript code. The code is more reliable and less prone to errors via static analysis at build time.

The following is an example that combines all of the concepts described above:

<script setup lang="ts">
  import { ref, onMounted } from 'vue'
  import axios from 'axios'

  interface Project {
    'projects/id': string
    'projects/name': string
    'projects/description': string
    'projects/tags': string[]
  }

  const projects = ref<Project[]>([])

  const getProjects = async () => {
    await axios
      .get<Project[]>('/api/projects')
      .then(async (response) => {
        if (response.status !== 200) {
          return
        }
        const data = await response.data
        projects.value = data
      })
      .catch((error) => {
        console.error(error)
      })
  }

  onMounted(async () => {
    await getProjects()
  })
</script>

<template>
  <v-list>
    <v-list-item v-for="(project, index) in projects">
      <v-list-item-title>
        {{ project['projects/name'] }}
      </ v-list-item-title>
    </ v-list-item>
  </ v-list>
</ template>

The Composition API, the lifecycle hooks, and Typescript support, provide a really smooth transition from React to Vue. I found extremely easy to approach Vue, with a prior experience working on React web applications.

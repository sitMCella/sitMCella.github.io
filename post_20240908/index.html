Catalog of Turing Machines

Sep 8, 2024

Some years ago I read the book "The annotated Turing" by Charles Petzold, and I decided to implement in a web application some of the machines designed by Turing.

The original paper from Turing created the basis for the theory of computation, and introduced the concept of Turing Machine. Turing created a machine that, starting from basic operations, was able to produce computable numbers.

Turing started to describe the concept of m-configurations as the finite number of configurations (states) in a machine. Turing also added a physical structure to the machine with the concept of "tape", a one-dimensional tape divided into squares. These squares can contain a symbol, and the machine can read from and write to the squares of the tape.

A limitation imposed by Turing is that the machine can only scan one square at a time. The machine switches from one m-configuration to another depending on the scanned symbol. The m-configurations can be used to retain information about the symbols that have been scanned previously.

At any moment in time the configuration of the Turing Machine is determined by the m-configuration and the scanned symbol. The operations available on the machine are: print (P), erase (E), move right (R), move left (L). The machine prints the symbols 0 and 1, and these symbols constitute the computed sequence in the tape. It's important to mention that Turing in his paper describes machines that require the tape to extend just on the right direction.

The following is the first machine defined by Turing in his paper:

m-configsymboloperationsfinal m-config
bNoneP0,Rc
cNoneRe
eNoneP1,Rf
fNoneRb

Compared to the original paper, I used the latin letters instead of the German letters for the m-configurations.

In order for Turing to create advanced machines, he introduced the symbols schwa, and x. The symbol schwa is used as a sentinel, and in the Turing machines two schwa symbols at the beginning of the tape indicate the left boundary of the number. The symbols 0 and 1 are always printed with a space of one square in between. The intermediate square is used to print the symbol x when required by the machine, and its used as a sort of temporary memory.

I created a web application that implements some of the machines described by Turing in his paper. The application retains the concept of tape, m-configuration, symbol, and operations. I used latin letters in the machine configuration and the tape, and I replaced the symbol schwa with the letter "s".

Starting from the current m-configuration and the scanned symbol, at each stage the application executes the operations listed in the current m-configuration and moves to the final m-configuration. I made the design choice to print the complete configuration of the machine in a new line for each stage of execution, so the complete flow can be tracked from start to finish. Another design choice I made was to limit the total number of squares in the tape, defined by the user in the initial configuration, so the tape does not extend indefinitely.

Headers with CSS Position Sticky

Aug 18, 2024

I have recently worked on a story outlining web application. A story outline defines the storyline of a book. In a specific page of the application, I wanted to display the scenes of a book grouped by chapters and plots. A table format structure is a viable approach: the header row is used for collecting the chapters and the header column for collecting the plots.

The requirements of the application are the following: 1. the header row should always be visible on vertical scrolling, and 2. the header column should always be visible on horizonal scrolling.

I decided to use the CSS sticky position property to solve this specific problem.
The position properties in CSS are used to define the position of the elements in the page. The position: sticky property in CSS is a hybrid of the relative and fixed CSS positions. An element with position: sticky is treated as relatively positioned until it crosses a specified threshold (defined by top, right, bottom, or left), at which point it becomes fixed to that position in the viewport. The viewport represents the area of the web page that is currently visible. All the major browsers support the CSS position sticky element, you can check on caniuse.com.

I created a simple example using a static html page to explain how to make use of the CSS sticky position property for this use case.
The following is the CSS code.

<style>
.project-map-container {
  width: 1500px;
  height: 600px;
  overflow-y: auto;
}

.project-map-corner {
  width: 210px;
  height: 40px;
  position: absolute;
  z-index: 10;
  top: 8px;
  left: 8px;
  background-color: #d3dce6;
}

.chapters-header-container {
  position: sticky;
  z-index: 1;
  top: 0px;
  background-color: #d3dce6;
  display: inline-flex;
}

.empty-chapter {
  padding-top: 10px;
  padding-bottom: 10px;
  margin: 10px;
  width: 200px;
  max-width: 200px;
  min-width: 200px;
}

.chapter-box {
  padding-top: 10;
  padding-bottom: 10px;
  margin-left: 10px;
  margin-right: 10px;
  margin-top: 10px;
  margin-bottom: 0px;
  width: 450px;
  max-width: 450px;
  min-width: 450px;
  justify-content: center;
  align-items: center;
}

.plot-row {
  display: flex;
}

.plot-container {
  padding-top: 10px;
  padding-bottom: 10px;
  padding-left: 10px;
  margin-right: 10px;
  margin-top: 0px;
  margin-bottom: 0px;
  width: 200px;
  max-width: 200px;
  min-width: 200px;
  display: grid;
  position: sticky;
  left: 0px;
  z-index: 1;
  background-color: #d3dce6;
}

.plot-container-overlay {
  margin-left: 0px;
  position: fixed;
  left: 0px;
  z-index: 1;
  height: 100%;
  width: 210px;
  background-color: #ffffff;
}

.scenes-container {
  padding-top: 0px;
  padding-bottom: 0px;
  margin-left: 10px;
  margin-right: 10px;
  margin-top: 0px;
  margin-bottom: 0px;
  width: 450px;
  max-width: 450px;
  min-width: 450px;
  min-height: 100%;
  justify-content: center;
  align-items: center;
}

.scenes-box {
  min-height: 200px;
  margin-top: 10px;
}
</style>

The CSS sticky position property is applied to the DIV elements that contain the chapters and the plots. I have also added overlay DIVs to cover enterely the header row and the header column. The z-index property is used to keep the header row and the header column on the front. In addition, I have added a DIV on the corner to hide the chapters on horizontal scrolling, and to hide the plots on vertical scrolling.
